\chapter{Benutzeroberfläche}
Um die Daten des Monitoring Knotens sinnvoll darstellen zu können, soll eine
graphische Oberfläche existieren, welche diese Daten übersichtlich darstellt.
Hierbei gilt es, die bereits vorhandene rqt-Gui um weitere Funktionalität zu
ergänzen, sei es in Form von neuen Widgets oder durch Modifikationen bereits
Existierender.


\section{Merkmale der graphischen Benutzeroberfläche}
\begin{itemize}
	\item Implementiert mit Hilfe von PyQt\footnote{\hyperlink{http://www.riverbankcomputing.co.uk/software/pyqt}{http://www.riverbankcomputing.co.uk/software/pyqt}} bzw. als rqt\footnote{\hyperlink{http://wiki.ros.org/rqt}{http://wiki.ros.org/rqt}} Widget
	\item Modularer Aufbau der Benutzeroberfläche in Form von Widgets, welche in die bereits bestehende rqt-Oberfläche integriert werden können
	\item Erweiterung der Funktionalität einiger rqt Widgets, um zusätzliche Informationen anzeigen zu können
	\item Verwendung geeigneter graphischer Bibliotheken, um eine gute Übersichtlichkeit der Daten zu gewährleisten. Erwähnenswert ist hier insbesondere Graphviz\footnote{\hyperlink{http://www.graphviz.org/}{http://www.graphviz.org/}} für den Graphen der Hosts/Nodes und Matplotlib\footnote{\hyperlink{http://matplotlib.org/}{http://matplotlib.org/}} für die Darstellung des zeitlichen Verlaufs von beispielsweise des Netzwerk-Traffics oder der CPU-Auslastung
	
\end{itemize}

\section{Protoypen/Skizzen}
\subsection{ÜbersichtsWidget}
	Den Kern der graphischen Oberfläche bildet das ÜbersichtsWidget, welches einen Großteil der relevanten Informationen auf einer kleinen und übersichtlichen Oberfläche zusammenfasst. Hierunter fallen unter anderem Information zum aktuellen Status aller Knoten, Fehlermeldungen und Grafiken zum zeitlichen Verlauf der Auslastung unterschiedlicher Komponenten.
	\vspace{0.5cm}
	
	\begin{figure}[H]
	\begin{center}
	\includegraphics[width=0.6\linewidth]{gui/uebersichtsgui_final_information.png}
	\caption{ÜbersichtsWidget - Reiter Information}
	\end{center}
	Hier werden allgemeine Informationen, d.h. der Durchschnitt der Auslastung aller Knoten angezeigt und durch die grüne "Ampel" eine korrekte Funktion aller (wichtigen) Knoten signalisiert.
	\end{figure}
	
	\begin{figure}[H]
	\begin{center}
	\includegraphics[width=0.7\linewidth]{gui/uebersichtswidget_final_progress.png}
	\caption{ÜbersichtsWidget - Reiter Progress}	
	\end{center}
	Die Informationen des Informations-Tabs werden hier in Form von Graphen, d.h. im zeitlichen Verlauf dargestellt. Das betrachtete Intervall kann natürlich verändert werden.
	\end{figure}
	
	\begin{figure}[H]
	\begin{center}
	\includegraphics[width=0.7\linewidth]{gui/uebersichtswidget_final_error.png}
	\caption{ÜbersichtsWidget - Reiter Error}
	\end{center}
	
	Im Fehlerfall ändert sich die Farbe der "Ampel", sodass der Nutzer unmissverständlich erkennt, dass ein Problem existiert, das für eine korrekte Funktionsweise behoben werden muss. Im Error-Tab findet der Benutzer dann den Log der letzten Fehler, um so schnell und einfach herauszufinden, wo das Problem liegt.
	\end{figure}

\subsection{Auswahl Widget}
	Dieses Widget zeigt anhand des gerade ausgewählten Elements (z.B.durch Anklicken im Nodegraph oder durch Auswählen in einer Liste) detaillierte Information zu dem Element an. Element bedeutet in diesem Fall entweder Knoten oder Host.
	Ähnlich wie das Übersichts-Widget, ist auch dieses Widget in Tabs organisiert, wobei die Tabs Progress und Error ähnlich funktionieren, wie beim Übersichtswidget, weshalb deren Funktionalität hier nicht erneut betrachtet werden soll.
	\vspace{0.5cm}
	\begin{figure}[H]
	\begin{center}
	\includegraphics[width=0.45\linewidth]{gui/SelectionWidget_final_info.png}
	\caption{AuswahlWidget - Reiter Information}
	\end{center}
	Der Reiter Information bietet, wie der Name schon sagt, sehr umfangreiche Information über das spezifische, gerade ausgewählte Element. Diese können je nach Element (Knoten/Host) unterschiedlich sein, umfassen jedoch u.A. Bandbreite, Latenz, Name der Topics auf denen gesendet wird, CPU-Auslastung, Speicherauslastung und andere.
	\end{figure}
	
	\begin{figure}[H]
	\begin{center}
	\includegraphics[width=0.45\linewidth]{gui/SelectionWidget_final_actions.png}
	\caption{AuswahlWidget - Reiter Actions}	
	\end{center}
	Als optionales Feature können die Knoten im Problemfall neugestartet werden bzw. überhaupt gestartet werden. Dies erleichtert die Problembehebung erheblich, da diese Funktion aus der Ferne ausgeführt werden kann.
	\end{figure}

\subsection{Listen Widget}
Ein sehr einfaches Widget, welches dazu gedacht ist, die vorhandenen Hosts und Knoten übersichtlich anzuzeigen.

	\begin{figure}[H]
	\begin{center}
	\includegraphics[width=0.7\linewidth]{gui/ListWidget.PNG}
	\caption{List Widget - Hauptansicht}	
	\end{center}
	Um die Übersichtlichkeit zu erreichen werden die vorhandenen Daten zunächst hierarchisch angezeigt, was sich durch De/Aktivieren der Punkte "Show Nodes" und "Show Hosts" modifizieren lässt. Des Weiteren lassen sich die Elemente nach Suchbegriffen filtern und ermöglichen so ein schnelles Findes des gewünschtes Elements. Ein weiterer Filter ermöglicht das Anzeigen nur solcher Elemente, die derzeit in einem Fehler-Zustand sind. Eine Kombination der Filter ist natürlich möglich. 
	\end{figure}
